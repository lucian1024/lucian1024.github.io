## 前言

在[上一篇文章](/posts/%E9%B8%BF%E8%92%99JSON%E5%BA%8F%E5%88%97%E5%8C%96%E6%96%B9%E6%A1%88%E6%8E%A2%E7%B4%A2%EF%BC%9A%E4%BB%8Eclass-transformer%E5%88%B0TurboTransJSON/)中，我们将 class-transformer 移植到了鸿蒙平台。本文记录如何将其打包成 har 包并发布到远端仓库，供其他开发者使用。

---

## 一、发布流程概览

根据 [官方文档](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-publish)，发布一个鸿蒙三方库的整体步骤如下：

1. **注册账号**：在 [https://ohpm.openharmony.cn/](https://ohpm.openharmony.cn/) 注册并登录个人账号。
2. **生成公私钥**：在本地生成 RSA 密钥对，用于身份认证。
3. **配置私钥路径**：通过 ohpm 命令将私钥路径写入本地配置。
4. **配置发布码**：将个人中心生成的发布码写入本地配置。
5. **上传公钥**：将公钥内容粘贴到个人中心的公钥管理模块。
6. **打包 har**：将三方库打包成 `.har` 格式。
7. **发布**：执行 `ohpm publish` 命令完成发布。

官方文档对整体流程介绍较为清晰，但对**公私钥和发布码的具体配置方式**描述较少，下面详细说明。

---

## 二、公私钥与发布码配置

### 步骤 1：生成公私钥

在本地终端执行以下命令，生成 PEM 格式的 4096 位 RSA 密钥对：

```bash
ssh-keygen -m PEM -t RSA -b 4096 -f your-keypath
```

参数说明：
- `-m PEM`：密钥格式为 PEM
- `-t RSA`：使用 RSA 非对称加密算法
- `-b 4096`：密钥长度 4096 位
- `-f your-keypath`：指定密钥文件输出路径（如 `~/.ssh/ohpm_key`）

> ⚠️ **注意**：OHPM 只支持加密密钥认证，生成时**必须输入 passphrase（密码）**，不能留空。  
> 命令执行后会生成两个文件：
> - `your-keypath`：私钥文件
> - `your-keypath.pub`：公钥文件

---

### 步骤 2：配置私钥路径

将私钥路径写入 ohpm 本地配置：

```bash
ohpm config set key_path your-keypath
```

将 `your-keypath` 替换为实际的私钥文件路径（绝对路径或相对路径均可）。

---

### 步骤 3：配置发布码

登录 [https://ohpm.openharmony.cn/](https://ohpm.openharmony.cn/) 个人中心，获取发布码（Publish ID），然后执行：

```bash
ohpm config set publish_id your-publishId
```

将 `your-publishId` 替换为个人中心显示的发布码字符串。

---

### 步骤 4：上传公钥

在 OHPM 个人中心的 **公钥管理** 模块，点击「添加公钥」，将本地公钥文件的内容粘贴进去：

```bash
cat your-keypath.pub
```

复制输出的全部内容，粘贴到公钥输入框中并保存。至此，身份认证配置完成。

---

## 三、将 TypeScript 三方库打包成 har 包

部分三方库（如 class-transformer）的源码是 TypeScript，官方提供了一个 [转换工具](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-convert)，但实际使用中存在**无法本地转换**、**转换产物有问题**等问题。

har 包本质上是一个 **tar.gz 压缩包**，只要目录结构和 `oh-package.json5` 配置正确，直接压缩即可。下面是完整的手动打包流程：

### a. 创建 package 目录

```bash
mkdir package
```

在 `package/` 目录下创建以下文件：

- `CHANGELOG.md`：变更日志
- `LICENSE`：开源协议文件
- `README.md`：库的说明文档

### b. 编译 TypeScript 工程，拷贝产物

```bash
# 编译 TypeScript（以 tsc 为例）
npx tsc

# 将编译产物拷贝到 package/src 目录
cp -r dist/ package/src/
```

`package/src/` 目录下应包含：
- `cjs/`：CommonJS 格式的编译产物（`.js` 文件）
- `types/`：TypeScript 类型声明文件（`.d.ts` 文件）

### c. 创建 oh-package.json5

在 `package/` 目录下创建 `oh-package.json5`，配置 har 包的元信息：

```json5
{
  "name": "class-transformer",
  "version": "0.5.1",
  "description": "Proper decorator-based transformation / serialization / deserialization of plain javascript objects to class constructors",
  "author": "TypeStack contributors",
  "license": "MIT",
  "main": "./src/cjs/index.js",
  "types": "./src/types/index.d.ts",
  "repository": "https://github.com/typestack/class-transformer.git",
  "compatibleSdkVersion": "5.1.0(18)",
  "obfuscated": false
}
```

字段说明：
- `name`：包名，需与 OHPM 上注册的包名一致
- `version`：版本号，遵循 semver 规范
- `main`：入口文件路径（相对于 package 根目录）
- `types`：TypeScript 类型声明入口
- `compatibleSdkVersion`：兼容的鸿蒙 SDK 版本
- `obfuscated`：是否对源码进行混淆，调试阶段建议设为 `false`

### d. 打包成 har 文件

```bash
tar -cvzf class-transformer.har package
```

> `tar -cvzf` 将 `package/` 目录压缩为 `class-transformer.har`，这就是可以发布的 har 包。

---

## 四、发布

打包完成后，执行发布命令：

```bash
ohpm publish class-transformer.har
```

如需发布到私有仓库，添加 `--publish_registry` 参数：

```bash
ohpm publish --publish_registry https://your-private-registry-url class-transformer.har
```

发布成功后，可在对应仓库中搜索到该包。
