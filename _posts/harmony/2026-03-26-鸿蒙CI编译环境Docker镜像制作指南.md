CI 构建鸿蒙应用需要具备鸿蒙编译环境的 Docker 镜像，本文介绍如何从零制作一个可用于 CI 构建的鸿蒙编译环境镜像。

## 1. 安装 Docker

下载并安装 [Docker Desktop](https://www.docker.com/)。

> **注意**：Docker 在某些环境下存在合规风险，如果 Docker 不可用，可使用 [Podman](https://podman.io/) 替代，其命令行接口与 Docker 基本兼容，将下文中的 `docker` 命令替换为 `podman` 即可。

## 2. 创建容器

启动 Docker 后，便可在命令行中使用。

### 2.1 选择系统镜像

编译鸿蒙需要 Linux 系统环境，在 Docker Hub 上选择 [tencentos3](https://hub.docker.com/r/tencentos/tencentos3) 作为基础镜像。

### 2.2 使用镜像创建容器

```bash
# 使用镜像创建容器
# --name: 容器名
# --platform: 镜像系统架构，鸿蒙构建工具只支持 x86_64 架构，因此必须指定 linux/amd64
# -it: 创建并启动容器，启动后命令行可直接操作容器
$ docker run --name tencentos3-harmony --platform linux/amd64 -it tencentos/tencentos3:latest
```

可通过命令行查看当前运行中的容器：

```bash
$ docker ps
CONTAINER ID   IMAGE          COMMAND       CREATED        STATUS         PORTS     NAMES
0bc004d4d05c   85a09c02c400   "/bin/bash"   20 hours ago   Up 3 minutes             tencentos3-harmony
```

### 2.3 其他常用操作

进入容器后，执行 `exit` 命令退出；下次启动已有容器：

```bash
# 0bc004d4d05c 是容器 ID
$ docker start -i 0bc004d4d05c
```

## 3. 配置编译环境

参考[华为官方文档](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/ide-command-line-building-app-V5)安装和配置所需编译环境，以下是几个需要特别注意的地方。

### 3.1 Java SDK

Oracle Java SDK 需要收费且公司禁用，推荐使用 [TencentKona-17](https://github.com/Tencent/TencentKona-17)。

若在容器内下载不便，可在宿主机下载后通过 `docker cp` 拷贝进容器：

```bash
# 拷贝 TencentKona 到容器内 /root/java 目录
$ docker cp Downloads/TencentKona-17.0.17.b1-jdk_linux-aarch64.tar.gz 0bc004d4d05c:/java

# 拷贝鸿蒙 commandline-tools 到容器内 /root/hmos 目录
$ docker cp Downloads/commandline-tools-linux-x64-6.0.0.868.zip 0bc004d4d05c:/hmos
```

### 3.2 配置 npm 镜像源

如果出现组件下载失败的情况，需要配置 npm 镜像源：

```bash
# 配置 npm 镜像源
[root@0bc004d4d05c ~]# npm config set registry https://<your-npm-registry>/
# 避免 SSL 校验失败
[root@0bc004d4d05c ~]# npm config set strict-ssl false
# 移除代理配置（代理配置会导致 npm 操作失败）
[root@0bc004d4d05c ~]# npm config delete proxy
[root@0bc004d4d05c ~]# npm config delete https-proxy
# 确认配置，结果中不应出现 proxy 相关配置
[root@0bc004d4d05c ~]# npm config list
```

### 3.3 安装 libGL 库

官方文档中的 `libgl1-mesa-dev` 在 TencentOS 下无法安装，替换为：

```bash
[root@0bc004d4d05c ~]# yum install -y mesa-libGL-devel
```

### 3.4 安装 Git

建议在容器内安装 Git，方便后续调试和代码管理：

```bash
[root@0bc004d4d05c ~]# yum install -y git
```

## 4. 生成镜像

容器环境安装和配置完毕后，退出容器，将其打包成镜像：

```bash
# 使用 0bc004d4d05c 容器生成名为 tencentos3-harmony 的镜像
$ docker commit 0bc004d4d05c tencentos3-harmony
```

可使用以下命令查看当前的镜像列表：

```bash
$ docker images
```

## 5. 上传镜像至 Docker Hub

生成的镜像可上传至 Docker Hub，供 CI 流水线拉取使用。

**Step 1**：在 Docker Hub 个人 Repositories 中创建一个 repo。

**Step 2**：为镜像创建 tag：

```bash
# tencentos3-harmony 是镜像名，{repository-name} 替换为刚创建的 repo 名
# 6.0.0.868 对应鸿蒙 commandline-tools 版本号，便于区分
$ docker tag tencentos3-harmony {repository-name}:6.0.0.868
```

**Step 3**：推送到 Docker Hub：

```bash
$ docker push {repository-name}:6.0.0.868
```

上传成功后，便可在 CI 流水线中通过镜像名拉取使用，进行鸿蒙应用的自动化构建。
