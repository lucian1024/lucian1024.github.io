本文记录一下开发中常用的Git命令。

## 1 常用Git命令简写别名

```
alias gd="git diff ."
alias gs="git status ."
alias gl="git log"
alias gl.="git log ."
alias ga="git add ."
alias gck="git checkout ."
alias gcl="git clean -df"
alias gb="git branch -av"

#mac平台重置仓库秘钥,注意缩进按如下格式
repass() {
    git credential-osxkeychain erase <<EOF
host=git.woa.com
protocol=https
EOF
}
```

## 2 Git修改账号邮箱

### 2.1 查看账号邮箱

```
git config --list
```

如果在非Git仓库下执行，展示的是全局的账号邮箱; 如果在Git仓库下执行，展示全局和当前仓库的账号邮箱，在前面的是全局的，在后面的是当前仓库的，如果二者一样，则只显示一个。

### 2.2 设置账号邮箱

```
# 设置全局 []内容替换为实际值
git config --global user.name [username]
git config --global user.email [email]

# 设置仓库，切换到仓库目录
git config user.name [username]
git config user.email [email]
```

## 3 分支合流

```
# 同步所有分支信息到本地
git fatch --all
# 切换到需要合流的分支
git checkout [mergeBranch]
# 确保分支更到最新
git pull origin [mergeBranch]
# 切换当前分支
git checkout [branch]
# 确保分支更到最新
git pull origin [branch]
# 合流
git merge --no-ff [mergeBranch]
```

## 4 修改提交记录

### 4.1 修改上一次的提交记录

```
# 修改提交记录
git commit --amend
# 修改提交人员 注意<>不能删除
git commit --amend --author="username <email>"
```

### 4.2 修改前几次的提交记录

```
# 例如修改第三个提交记录, HEAD~3表示操作HEAD到第三个的提交记录，改命令会展示3个提交记录，将需要修改的提交记录前的pick改成edit
git rebase -i HEAD~3
# 修改提交记录
git commit --amend
# 或者修改提交人员 注意<>不能删除
git commit --amend --author="username <email>"
# 继续rebase, 重复直到rebase结束
git rebase --continue
```

### 4.3 提交修改

```
# 如果commit还未提交到远端，执行如下命令提交
git push origin [branch]
# 如果commit已经提交到远端，需要增加-f参数强制提交
git push -f origin [branch]
```

## 5 撤销本地未提交Commit

```
# i表示要撤销多少个commit，只是测小Commit记录，代码仍保留
git reset --soft HEAD~i
```

## 6 忽略已经提交的文件

.gitignore添加文件路径，另外命令行执行

```
git rm -r --cache [文件名]
```

## 7 Mac终端git status 中文显示乱码

```
# 执行以下命令后显示正常
git config --global core.quotepath false
```

## 8 Mac终端git log等退出后结果未保留

```
# 如果遇到git log命令退出后结果未保留在终端(类似vim的表现)，执行以下命令解决:
git config --global --replace-all core.pager "less -F -X"
```
